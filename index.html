Prerequisites

  • Authorizations: Access to Fiori Launchpad Designer and PFCG.
  • System: SAP S/4HANA or SAP Fiori for SAP ERP with FLP.
  • Target: An SAP GUI transaction code (example: VA03).

Step 1 ? Open the Launchpad Designer

Use the Fiori Launchpad Designer to configure catalogs, tiles, and target mappings. In most on-prem setups:

  • For client-specific customizing: go to /ui2/flpd_cust
  • For cross-client configuration: go to /ui2/flpd_conf

Step 2 ? Create a Catalog

  1. In the left pane, select Catalogs.
  2. Click + (Create Catalog).
  3. Enter Title (e.g., Z_SALES) and optional ID and Description.
  4. Save.

Step 3 ? Create a Static Tile

  1. Open the catalog you created.
  2. Click Add ? App Launcher - Static.
  3. Fill Title, Subtitle (optional), Keywords, and Icon.
  4. In Navigation, provide the Semantic Object and Action that will link to your target mapping (configured in the next step). Example: Semantic Object = SalesOrder, Action = display.
  5. Save.

Step 4 ? Create Target Mapping to the TCode

The target mapping connects your tile?s intent (Semantic Object + Action) to the runtime app. For SAP GUI transactions, use Application Type: SAP GUI for HTML and specify the Transaction.

  1. In the same catalog, open Target Mappings and click +.
  2. Enter Semantic Object and Action to match the tile (e.g., SalesOrder / display).
  3. Set Application Type = SAP GUI for HTML.
  4. Set Transaction = your tcode (e.g., VA03).
  5. Save.
Tip: The tile and the target mapping must share the same Semantic Object and Action so the intent resolves correctly.

Step 5 ? Create Space and Page

Spaces and Pages are the modern replacement for Groups. Create a Space, then a Page, and add your catalog tile to that Page.

  1. In Designer, switch to Spaces and create a new Space (e.g., Z_SALES_SPACE).
  2. Create a Page (e.g., Z_SALES_PAGE) under that Space.
  3. In the Page builder, add a section and drag your tile from the catalog content.
  4. Save & Publish the Page.

Step 6 ? Assign Catalog and Space to a Role (PFCG)

Use PFCG to create/maintain a role that exposes the catalog and the space/page to users.

  1. Open PFCG, create a role (e.g., Z_SALES_FIORI).
  2. On the Menu tab, add Fiori Tile Catalog and select your catalog.
  3. Add Fiori Space and select your space/page.
  4. Generate authorizations and save the role.

Step 7 ? Assign Role to User

Assign the role to the end user (e.g., via SU01). After assignment, the user should see the new tile in the FLP after refresh/relogin.

Step 8 ? Test the Tile

  1. Log into the Fiori Launchpad as the test user.
  2. Open the Space and Page where you added the tile.
  3. Click the tile ? it should open the transaction in SAP GUI for HTML.

Troubleshooting

  • Tile not visible: Ensure the role includes the catalog and the space/page, and the user has the role.
  • Click shows error: Verify the target mapping?s Semantic Object and Action match the tile. Confirm Application Type is SAP GUI for HTML and the transaction code is correct.
  • Authorization issues: Generate authorizations in PFCG and check S authority objects for SAP GUI for HTML access.
  • Using Groups (legacy): If Spaces/Pages are not used, add the tile to a Group and include the Group in the role.
